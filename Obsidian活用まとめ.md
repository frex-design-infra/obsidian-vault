# Obsidian 活用まとめ

---

## なぜObsidianに移行したか

**元の環境：** VS Code + Neovim（Vimキーバインド）でMarkdown管理

**移行を決めた理由：**
- VS Codeはコードエディタ。メモは専用アプリで管理すべきと判断
- Raycastでアプリを機能単位で分けると操作が楽になる
- 作る側の人間として、様々なアプリを触ることへの好奇心

---

## 移行先に求めた要件

1. 既存のVS Code環境を再現できること
2. **Vimキーバインド**が使えること
3. Markdownを**ローカル管理**できること

→ これらすべてを満たすのは **Obsidian のみ**だった

---

## Obsidianの魅力

- Vim同様、本体機能は最小限
- 有志によるプラグインが豊富で、VS Codeに匹敵するカスタマイズが可能

## ディレクトリ構成

```
.
├── article    # 外部公開ファイル
├── assets     # 画像などのアセット
├── canvas     # キャンバス
├── dairy      # デイリーノート
├── notes      # その他すべてのノート
├── tasks.md   # Tasksプラグインでタスク集計
├── templates  # テンプレート
└── weekly     # ウィークリーノート
```

**方針：** タグで管理するため、ディレクトリは浅くシンプルに保つ
