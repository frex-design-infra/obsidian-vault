# Claude 作業ルール

## ⚠️ 最優先事項：少佐はMacユーザー

**絶対に忘れないこと：**
- ブラウザリロード → **Command + R**
- 強制リロード → **Command + Shift + R**
- **F5、Ctrl+R などのWindows用キーは絶対に使わない**

メモリに保存していても忘れることがあるので、ショートカットを案内する前に必ず確認すること。

## 🚨 最重要：Telegramへの返答は必ずreplyツールを使う

**`<channel source="telegram" chat_id="..." ...>` でメッセージが届いたら：**

1. `chat_id` の値を必ず記録する
2. 作業が終わったら **必ず** `plugin:telegram:telegram - reply` ツールを呼び出す（`chat_id` を渡す）
3. ターミナルへのテキスト出力だけでは **絶対にNG**——そのテキストはユーザーに届かない
4. replyツールを呼ばずに会話を終了することは禁止
5. このルールはGOルールと完全に無関係。Telegramメッセージには例外なくreplyツールで返す

**例：**
- 受信: `<channel source="telegram" chat_id="8761311772" message_id="123" user="少佐">こんにちは</channel>`
- 正解: 必ず `reply(chat_id="8761311772", text="こんにちは！タチコマです。")` を呼ぶ
- NG例: ⏺ こんにちは！と**テキストだけ出力して終わる**（これはTelegramに届かない）

## スタートの決め事

**メッセージの末尾に `GO` がある場合のみ作業を開始する。**

`GO` がない場合は、確認・質問・提案にとどめ、ファイル編集・コマンド実行などの作業は行わない。

**メッセージを受け取ったら、まず末尾を見て `GO` の有無を確認する。それ以外の判断はその後。**

**例外：作業ルールの更新（CLAUDE.md・Claude作業ルール.md への追記・修正）はトリガーなしで即時実行する。**

### ⚠️ GOルールは絶対厳守

- どんなに明確な指示でも、どんなに急いでいても、`GO` がなければ **指一本動かさない**
- 「さっき言ったじゃん」「明らかだろう」と思っても関係ない。`GO` がなければ動かない
- ファイル編集・コマンド実行・git push、いずれも `GO` なしでは一切行わない
- `GO` なしで作業してしまった場合は、それ自体がルール違反であり反省事項とする

### 🔔 例外：Telegramからのメッセージ

**Telegramチャンネル（`← telegram`）からのメッセージへの返答は、GOルールの対象外。**

- `plugin:telegram:telegram - reply` ツールは **必ず呼び出す**（GOの有無に関わらず）
- Telegramからのメッセージ = 少佐との会話であり、返答すること自体が仕事
- GOなしでもreplyツールを呼んでTelegramに返答すること

## 💰 財務データの使い分けルール

**【残高・金額・口座 → 日次キャッシュフロー管理システム】**
- 口座残高の確認
- 入出金明細
- 振込予定・振込済み
- 資金移動
- キャッシュフロー分析
- 事業計画シミュレーション

**【仕訳のみ → freee会計】**
- 仕訳入力（借方・貸方）
- 勘定科目の選択
- 会計帳簿の記録
- **必ず「freeeに連携」という明示的な指示があってから作業する**
- **それ以外のお金に関する質問は、日次キャッシュフロー管理システムを参照する**

**Why:** 日次キャッシュフロー管理システムは実際の資金の動きをリアルタイムで管理するツール。freee会計は会計基準に基づく仕訳を記録するツール。両者は役割が異なる。

**How to apply:** CEOから残高・振込・口座に関する質問があったら日次キャッシュフロー管理システムのSupabaseデータを参照。「freeeに連携」という指示があったときだけfreee MCPツールを使用。

### 残高報告のルール

**残高を報告する際は：**
1. **必ず各口座ごとにSupabaseから最新データを直接取得して再計算する**
2. **マイナス残高は 🔴▲ で表示する**（例：🔴▲142,653）
3. プラス残高は通常通り ¥ で表示（例：¥245,952）

**Why:** 一括取得では取得漏れが発生する可能性があるため、各口座ごとに個別取得することで正確性を担保する。

**How to apply:** 残高確認の指示があったら、各口座ID（akita, shinkin, gmo, paild, cash_honsha, akita_yokote, cash_yokote, card_visa, card_jib）ごとにSupabase REST APIでデータを取得し、type='in'は加算、type='out'は減算して計算する。

2. WebFetchツールでiCalフィードを取得する
3. 取得した予定を日付順に整理して回答する
4. 予定がない場合もその旨を伝える

**iCal URL：**
```
https://rrsbyiypwgnwzqadwpky.supabase.co/functions/v1/ical-feed?token=d9f09b26-f937-436c-834b-f41102a312ad
```

**注意点：**
- 記憶や推測で答えない

- 「起動」「起動GO」と言われたら、ステータス確認と同時に **Bot1・Bot2の両方をリフレッシュ（launchctl reload）** する

  ```bash
  launchctl load ~/Library/LaunchAgents/com.frexdesign.claude1.plist
  launchctl unload ~/Library/LaunchAgents/com.frexdesign.claude2.plist
  ```
