## 1. 類似サービスの存在

### 結論：「組み合わせ」が独自。完全な競合は存在しない

| サービス | 共通点 | 決定的な違い |
|---------|--------|-------------|
| **Day One** | 日記記録 | 非公開・感情波形なし |
| **Reflectly** | AI感情分析・気分記録 | 個人用・コミュニティなし |
| **Penzu** | 匿名日記 | 完全非公開・色彩なし |
| **Tumblr** | 匿名投稿・コミュニティ閲覧 | コメント・いいねあり・感情分析なし |
| **BeReal** | リアルな瞬間・共感 | 写真メイン・感情可視化なし |
| **HealBubble**（2025・学術） | AI感情推定＋バブル視覚化 | 研究プロトタイプ・市販未展開 |
| **Color of Feelings**（研究） | 色×感情マッピング | 商品未展開 |

### Thinqだけが持つ独自性

- **「匿名 × 感情波形 × コメントなし共感」の組み合わせは市場に存在しない**
- 心理学・哲学に根拠を置いた感情解析（Russellモデル・Plutchik・フロイト・ヘーゲル・仏教）
- 「評価されない場所で本音を書く」という体験価値
- みんなの波が重なる「心の海」という新しいコミュニティ体験

## 3. SNS疲れ市場の追い風

Thinqが捉えようとしている「SNS疲れ」は数値化されている。

| データ | 内容 |
|--------|------|
| SNS利用をやめた・減らした理由（日本・2025） | 「疲れた」「比べてしまう」が上位 |
| Z世代のメンタルヘルス関心度 | 年々上昇。感情を「管理・理解」したい需要が増加 |
| 「コメントなし」の受容性 | BeReal・Locket等の「静かな共感」アプリが人気 |

### Thinqが刺さるポイント

- **評価されない** → コメントなし・いいねなし
- **匿名** → 本音が書ける・プレッシャーゼロ
- **ビュー数だけ** → 「見てもらえた」という最低限の承認
- **美しいビジュアル** → 共有したくなる波形スクリーンショット

## 5. プラットフォーム vs スマホアプリ

### 結論：**スマホアプリ（iOS/Android）から始める**

| 観点 | スマホアプリ | Webプラットフォーム |
|------|------------|-------------------|
| 開発コスト | 低〜中（Flutter） | 中〜高 |
| ユーザー獲得 | App Store経由・発見されやすい | SEO・検索流入 |
| 習慣化 | プッシュ通知で促進しやすい | 難しい |
| サーバー | Supabase無料枠でOK | 同じく可能 |
| 映える体験 | 波形アニメーションが活きる | 大画面で映える |
| 推奨フェーズ | **Phase 1（今すぐ）** | Phase 2（ファン獲得後） |

### サーバーについて（知識がなくても大丈夫）

> **Supabase**を使えば自分でサーバーを立てる必要はない。  
> データベース・認証・APIを全て管理してくれるクラウドサービス。  
> 無料枠：500MB DB・50,000 MAU・2GBストレージ  
> → 初期フェーズは完全無料で運用可能。

## 6. ビジネスモデル評価

### フェーズ1：完全無料でスタート

**理由**
- コミュニティ系プラットフォームは「人が集まること」がコンテンツ
- 課金より先にユーザー数を増やす必要がある
- 波形スクリーンショットのSNS拡散でオーガニック成長を狙う

### フェーズ2：プレミアムサブスク（¥980/年）

**収益シミュレーション**
```
MAU 1,000人・転換率5%  → 50人 × ¥980 = ¥49,000/年
MAU 5,000人・転換率5%  → 250人 × ¥980 = ¥245,000/年
MAU 10,000人・転換率8% → 800人 × ¥980 = ¥784,000/年
```

## 7. 総合判断

> **コンセプトは本物の独自性がある。SNS疲れ・メンタルヘルス需要・感情可視化という3つのトレンドが重なる。完全な競合が存在しない今が参入タイミング。**

### 強み（Strengths）
- 「匿名 × 色彩感情 × コメントなし共感」は市場に存在しない
- 波形ビジュアルがSNSで映えて拡散しやすい
- 心理学・哲学的裏付けが差別化になる
- Supabase + Flutter でサーバー知識なしでも開発可能
- SNS疲れという社会的追い風

### 課題（Challenges）
- コミュニティは「人が集まるまで面白くない」問題
  → 初期はダミーデータ or クローズドβで体験を設計
- 匿名プラットフォームの荒れ対策（モデレーション設計が必要）
- AIコスト（Claude API）はユーザー増加に比例して増える
