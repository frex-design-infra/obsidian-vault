# Thinq — 感情分析設計

> テキストから「心の色と波形」を生成するAI設計ドキュメント

← コンセプト全体は [[Thinq 01 - アプリコンセプト]] へ

## 投稿スタイルは2種類、でも「波」は同じ

Thinqへの投稿は長さを問わない。  
1文字のつぶやきも、1000文字の日記も、**同じ「波」として流れる。**

| スタイル | 文字数の目安 | 例 |
|---------|------------|-----|
| **つぶやき** | 1〜100文字 | 「疲れた」「なんかいいことあった気がする」「。」 |
| **日記・メモ** | 100〜1000文字 | 今日あったこと、感じたこと、考えたこと |

### 波形への影響

```
つぶやき（短い）            日記・メモ（長い）
    │                           │
シンプルな波形               複雑で豊かな波形
色は1〜2色                  色が混ざり合う
波の幅が細い                波の幅が広い
```

## 感情分析のアーキテクチャ

```
ユーザー投稿（テキスト）
        │
        ▼
  文字数判定
  ┌─────────────────────────────────┐
  │ 短い（〜100文字）                │
  │   → シンプル分析モード           │
  │ 長い（100文字〜）                │
  │   → ディープ分析モード           │
  └─────────────────────────────────┘
        │
        ▼
  Claude API（感情分析プロンプト）
        │
        ▼
  分析結果（JSON）
  ┌─────────────────────────────────┐
  │ valence      -1.0 〜 +1.0       │  快 ↔ 不快
  │ arousal       0.0 〜 1.0        │  低覚醒 ↔ 高覚醒
  │ intensity     0.0 〜 1.0        │  感情の強さ
  │ emotions[]   主要感情（最大3つ）  │
  │ ambiguity    0.0 〜 1.0         │  曖昧さ・不確かさ
  └─────────────────────────────────┘
        │
        ▼
  波形パラメータ変換
        │
        ▼
  ビジュアル生成（Flutter）
```

## Claude API プロンプト設計

### システムプロンプト

```
あなたは心理学・哲学に精通した感情分析AIです。
ユーザーの投稿テキストから感情状態を読み取り、
構造化されたJSONを返してください。

# 分析の根拠となる理論
- Russellの感情環状モデル（valence × arousal）
- Plutchikの感情の輪（8基本感情）
- フロイトの防衛機制（抑圧・投影・合理化など）
- ヘーゲルの弁証法（対立する感情の共存）
- 仏教・ストア哲学（受容・執着・手放し）
- 認知的不協和理論（葛藤・矛盾の検出）

# 重要な注意点
- 日本語の曖昧な表現・口語・SNS表現を深く理解すること
- 「疲れた」は身体的疲労か感情的消耗か文脈から判断する
- 「やばい」「草」などのスラングの感情的ニュアンスを理解する
- 「。」や「…」だけの投稿にも感情を読み取ること
- 表面的な言葉だけでなく、言葉の「重さ」「温度」を感じ取ること

# 出力形式（JSON）
{
  "valence": -1.0〜1.0,        // 快(+) ↔ 不快(-)
  "arousal": 0.0〜1.0,          // 低覚醒(0) ↔ 高覚醒(1)
  "intensity": 0.0〜1.0,        // 感情の強さ
  "ambiguity": 0.0〜1.0,        // 曖昧さ・不確かさ（高いほど波が霞む）
  "emotions": [                 // 主要感情（最大3つ、強い順）
    {
      "label": "穏やか",
      "weight": 0.6,
      "color_hint": "teal"
    }
  ],
  "wave_note": "一言で表す波の特徴（日本語）"  // 例：「静かに沈んでいく波」
}
```

## 感情ごとの波形パラメータ変換

| 感情 | valence | arousal | 波の高さ | 波の幅 | 周波数 | 色 |
|------|---------|---------|---------|--------|-------|-----|
| 怒り・激情 | -0.8〜-0.5 | 0.8〜1.0 | 高い | 細い | 高速 | コーラル〜レッド |
| 不安・緊張 | -0.6〜-0.3 | 0.6〜0.9 | 中・細かく震える | 細い | 不規則 | ローズ〜マゼンタ |
| ワクワク・高揚 | +0.5〜+1.0 | 0.7〜1.0 | 高い | 広い | 中速 | アンバー〜イエロー |
| 喜び・感謝 | +0.6〜+1.0 | 0.3〜0.6 | 中 | 広い | 遅い | グリーン〜ライム |
| 穏やか・安心 | +0.3〜+0.7 | 0.0〜0.3 | 低い | 広い | ゆっくり | ティール〜ミント |
| 悲しみ・孤独 | -0.7〜-0.4 | 0.1〜0.4 | 低・深く沈む | 中 | 遅い | ブルー〜インディゴ |
| モヤモヤ・疲れ | -0.3〜0.0 | 0.0〜0.3 | 低い | 広い | とても遅い | スチール〜グレー |
| 複雑・混乱 | -0.2〜+0.2 | 0.4〜0.7 | 不規則 | 中 | 不規則 | バイオレット〜ラベンダー |

### ambiguity（曖昧さ）の扱い

```
ambiguity が高い（0.7〜）  → 波の透明度が上がる（半透明・霞がかった表現）
ambiguity が低い（〜0.3）  → 波がくっきり・鮮明に表示される
```

## 1日複数回投稿の設計

### 各投稿は独立した波として流れる

```
タイムライン（心の海）:

  10:23  ~~~（スチール）~~~ 「眠い」
  12:45  ///（アンバー）/// 「ランチが美味しかった」
  18:02  ---（コーラル）--- 「上司にまた言われた」
  22:30  ~~~（ティール）~~~ 「お風呂入ったらちょっとマシになった」
```

→ 1日の感情の変化が「波の流れ」として見える

### 自分の履歴画面での表示

```
4月16日（木）   4投稿

  10:23 〜（スチール）    「眠い」
  12:45 〜（アンバー）    「ランチが美味しかった」
  18:02 〜（コーラル）    「上司にまた言われた」
  22:30 〜（ティール）    「お風呂入ったらちょっとマシになった」

  ── 今日の1日波形（4つを合成） ──
  [複合波形がグラデーションで表示される]
```

**1日の合成波形**：その日の全投稿を合算した「今日の波」が最後に生成される

## 日本語特有の表現対応表

| 表現 | 意味の解釈 | valence | arousal |
|------|-----------|---------|---------|
| 「草」「草生えた」 | 笑い・可笑しい | +0.6 | 0.6 |
| 「やばい（ポジ）」 | 驚き・興奮 | +0.5 | 0.8 |
| 「やばい（ネガ）」 | 困惑・焦り | -0.5 | 0.7 |
| 「しんどい」 | 身体的・感情的疲労 | -0.6 | 0.2 |
| 「えぐい」 | 強い感情（文脈依存） | ±0.5 | 0.7 |
| 「なんか…」 | 言語化できない感情 | ambiguity↑ | - |
| 「ちょっと…」 | 抑制・遠慮 | ambiguity↑ | - |
| 「…（省略）」 | 言葉にならない感情 | ambiguity最大 | - |
| 「はあ」 | 疲れ・ため息 | -0.4 | 0.1 |
| 「たのしかった」（ひらがな） | 柔らかい喜び | +0.5 | 0.4 |

## エラー・エッジケース対応

| ケース | 対応 |
|--------|------|
| 空欄投稿 | 投稿ボタンを無効化 |
| スペースのみ | 投稿ボタンを無効化 |
| 絵文字のみ | 絵文字の感情的意味で分析 |
| 外国語混じり | 日本語部分を優先、全体の雰囲気も考慮 |
| API タイムアウト | デフォルト波形（グレー・曖昧）で投稿 |
| 攻撃的・有害なテキスト | 波形生成はするが、モデレーション検討 |

## APIコスト見積もり

```
Claude API（claude-3-haiku 使用を推奨）
  入力: ~200トークン（システムプロンプト + ユーザーテキスト）
  出力: ~150トークン（JSON）

コスト: 約 $0.0003/回（haiku）

シミュレーション:
  MAU 1,000人 × 平均3投稿/日 × 30日 = 90,000回/月
  → 月 $27 程度（初期フェーズは十分許容範囲）
```
